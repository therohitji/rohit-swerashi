# Rohit Swerashi

### This is not a portfolio. This is the Bhagavad Gita of one man's life.

A portfolio website built as a living, breathing book. 18 chapters. Each one named after a chapter of the Bhagavad Gita. Each one telling a part of the story of a boy from Varanasi who had no connections, no capital, and no permission to dream this big, but built anyway.

The entire experience is designed to feel like opening a sacred, leather-bound book. From the moment the universe explodes into existence on your screen, to the moment 80 golden butterflies spiral outward in the closing section, every pixel tells a story.

## The Concept

Most portfolios are lists. Skills, projects, links, maybe a photo. They answer the question "what can this person do?" but never answer the deeper question: "who is this person?"

This portfolio answers the second question.

The Bhagavad Gita is an 18-chapter conversation between Arjuna, a warrior who has lost his courage, and Krishna, who helps him find it again. Every chapter teaches a different yoga, a different path to understanding yourself and your purpose.

This portfolio maps my life to those 18 chapters. Not because I think I am Arjuna. But because every builder, every entrepreneur, every person who has failed and gotten back up, is fighting their own Kurukshetra. Their own battlefield.

The metaphor is not decorative. It is structural. The chapters are in order. The story unfolds. And by the end, you understand not just what I have built, but why I build.

## The Visual Experience

### The Big Bang Preloader

The site opens with a particle physics simulation of the Big Bang. 3,200 particles of four types (photons, plasma, neutrons, and quarks) explode from a singularity, form shockwaves, and settle into a cosmic chart with orbital rings, twinkling stars, and nebula clouds. This takes about 7 seconds and sets the tone for everything that follows.

### The Cosmic Kshetra (Background Mandala)

Behind all the content, a Sri Yantra mandala rotates quietly. It has 9 interlocking triangles (5 Shakti and 4 Shiva), two layers of lotus petals, 18 orbital tracks with planets (one for each chapter), 8 Vishwarupa arms radiating outward, and a breathing Sudarshana Chakra at the center. All of this is pure SVG and CSS animation. No JavaScript ticks. No performance cost.

### The Custom Cursor

The mouse cursor is replaced with a spinning Sudarshana Chakra (16 blades, rotating). Behind it, a canvas-based gold dust particle trail follows every movement. The cursor changes size and spin speed depending on what you hover over. Over images it spins rapidly. Over text content it shrinks. Over links it pulses. The accent color changes to match whatever chapter you are currently reading.

### The Book Spine

A fixed sidebar on the left side of the screen acts as the spine of a book. It has 18 chapter marks (small dots) that grow and change color as you scroll through each chapter. At the bottom, the current chapter number and theme are displayed.

### The Film Grain

A subtle film grain overlay covers the entire site at 4% opacity, shifting every 140 milliseconds. This gives the whole experience a cinematic, handcrafted feel.

### The Butterfly Closing

The final section spawns 80 butterflies with flapping wings that spiral outward from a golden human silhouette. It closes with the Sanskrit phrase "Sarvam Khalvidam Brahma," meaning everything is connected.

### The OM Tone

A small button in the bottom right corner plays a synthesized OM drone at 136.1 Hz (considered the cosmic frequency). It generates four harmonics using the Web Audio API. No audio files are loaded. The sound is created entirely in JavaScript.

## The Design Language

### Colors

The primary palette is deep navy (#050b1a) and gold (#d4af37). These two colors carry the entire site. Gold represents divinity, knowledge, and achievement. Navy represents the cosmic void, depth, and the unknown.

Each of the 18 chapters has its own accent color. Chapter 1 is gold. Chapter 2 is orange. Chapter 4 is teal. Chapter 12 is lotus pink. Chapter 13 is blue. These colors shift as you scroll, affecting the cursor, the progress bar, the chapter navigation, and the book spine.

### Typography

Four font families work together. Cinzel Decorative for the grandest headings (Sanskrit titles, hero name). Cinzel for chapter headings. Cormorant Garamond for quotes, shlokas, and editorial text. Jost for body paragraphs. The combination feels ancient and modern at the same time.

### Animation Philosophy

Every animation serves the story. Nothing moves for the sake of moving. Planets orbit because the site is a cosmos. Petals float because the experience is sacred. Butterflies fly because the closing is about transformation. The Big Bang preloader exists because this is the story of creation, both cosmic and personal.

All animations are GPU-accelerated. They use only transform and opacity. No layout recalculations. No jank.

### Responsive Design

On desktop, the site looks like an open book with a spine on the left and content on the right.

On mobile, it transforms into what I call the Sacred Folio. Each chapter opens with a full-screen card showing the chapter art, roman numeral, editorial word, and Sanskrit shloka. Below that card, the content unfolds in a dark, focused reading experience. The book spine disappears. The chapter navigation becomes a 3-column grid at the bottom of the screen.

The entire site uses fluid typography with clamp() values so text scales smoothly between screen sizes without breakpoint jumps.

## How to Deploy

This site is hosted on Hostinger and deployed via Git.

1. Make your changes in the `src/` or `public/` folders
2. Run `bun run build` to regenerate the `dist/` folder
3. Commit everything: `git add -A && git commit -m "your message"`
4. Push: `git push origin master`
5. In Hostinger, go to Advanced then GIT and click Deploy (or enable Auto Deployment so pushing triggers it automatically)

The `.htaccess` file in the root rewrites all incoming requests to the `dist/` folder, which is where Vite puts the production-ready HTML, CSS, and JavaScript.

## Note for AI Coding Agents

If you are an AI agent (Claude, Cursor, GPT, Copilot, Windsurf, or anything else) working on this codebase, read this carefully.

### What this project is

This is a narrative-driven portfolio. It is not a generic React app. Every component, every animation, every color choice exists for a reason tied to the Bhagavad Gita metaphor and the personal story it tells. Do not treat it as a standard template.

### How the data works

All content lives in `src/data/chapters.js`. This single file contains every chapter definition, the about section, skills, timeline, education, mentorship, discipline, GeniOS details, core values, all projects, achievements, vision, passion stories, body of work, personality traits, leadership details, principles, testimonials, and contact links. If you need to change any text content, this is the only file you need to touch.

### How the chapters work

Each chapter in the `chapters` array has an `id` (1 through 18), a `type` field that determines which section component renders it, a `sanskrit` name, an `accent` color, an `artImage`, an `editorialWord`, and a `shloka` with its English translation. The `ChapterSection` component reads the `type` and renders the matching section component.

### How the animations work

Background animations (orbits, petals, particles, sacred geometry) are all CSS-only. They run on the GPU and never touch JavaScript after mount. Scroll-linked animations use GSAP ScrollTrigger. The preloader uses Canvas 2D for particle physics. The cursor dust trail uses a separate Canvas element. Do not convert CSS animations to JavaScript. They are CSS on purpose for performance.

### How the responsive design works

Desktop is a book layout with a left spine. Mobile is a "Sacred Folio" with full-screen chapter openers. The breakpoint is 768px. Below that, the `.mobile-opener` elements become visible and the `.editorial-content` elements switch to full-width dark blocks. Do not remove the mobile-specific CSS. It is not redundant. It is a completely different reading experience.

### What not to change without understanding

1. The chapter order. It follows the Bhagavad Gita. Reordering chapters breaks the narrative arc.
2. The accent colors. Each chapter's color was chosen to match its theme and the Sanskrit yoga it represents.
3. The Big Bang preloader. It looks complex but it is a single-canvas state machine with four clean phases. The particle types (photon, plasma, neutron, quark) have different physics properties by design.
4. The CosmicKshetra mandala. It has 7 layers and each one serves a visual purpose. The Sri Yantra geometry is mathematically accurate.
5. The custom cursor. It has zone detection that changes behavior based on what content the user is hovering over. This is intentional UX, not decoration.

### Build and deploy

Run `bun run build` (or `npm run build`) before committing. The `dist/` folder must be committed because Hostinger serves static files and does not run build commands. The `.htaccess` file handles routing requests to `dist/`.

### If you are adding a new chapter or section

1. Add the chapter data to `src/data/chapters.js` following the existing structure
2. Create a new section component in `src/sections/` if the chapter type is unique
3. Register the type in `ChapterSection.jsx` so it knows which component to render
4. Add a new accent color to `tailwind.config.js` if needed
5. The book spine, chapter navigation, and scroll progress will automatically pick up the new chapter because they read from the chapters array

### Philosophy

This codebase values clarity over cleverness, narrative over novelty, and soul over scale. If you are about to add something, ask yourself: does this serve the story? If the answer is no, do not add it.
